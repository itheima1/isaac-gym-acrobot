# Isaac Gym Envs 代码架构文档

本文档详细介绍了 `IsaacGymEnvs` 项目的代码结构和核心架构。该项目基于 NVIDIA Isaac Gym，提供了多个高性能的强化学习（RL）环境示例。

## 1. 项目概述

`IsaacGymEnvs` 是一个开源项目，旨在展示如何使用 NVIDIA Isaac Gym 进行大规模并行强化学习训练。它集成了 `rl_games` 库作为主要的 RL 训练框架，并使用 Hydra 进行灵活的配置管理。

## 3. 核心架构详解

### 3.1 训练入口 (`train.py`)

`isaacgymenvs/train.py` 是启动训练的主要入口。它的主要职责包括：
1.  **配置解析**：使用 Hydra 解析命令行参数和配置文件（`cfg/` 目录）。
2.  **环境注册**：将自定义的 Isaac Gym 环境注册到 `rl_games`。
3.  **算法注册**：注册自定义的算法（如 AMP）到 `rl_games`。
4.  **启动训练**：实例化 `rl_games.torch_runner.Runner` 并开始训练循环。

### 3.2 任务系统 (`isaacgymenvs/tasks`)

所有的环境任务都位于 `isaacgymenvs/tasks` 目录下。

*   **基类 (`VecTask`)**: 位于 `isaacgymenvs/tasks/base/vec_task.py`。所有的任务类都继承自 `VecTask`。它处理了与 Isaac Gym 模拟器的底层交互、缓冲区管理（Observation, Reward, Reset 等）以及多环境并行控制。
*   **任务实现**: 每个任务（如 `Ant`, `Cartpole`）通常对应一个 Python 文件。核心方法包括：
    *   `create_sim()`: 创建物理场景、加载资产、设置环境。
    *   `pre_physics_step(actions)`: 在物理步进前应用动作。
    *   `post_physics_step()`: 在物理步进后计算 Observation 和 Reward，处理 Reset。
*   **注册**: 在 `isaacgymenvs/tasks/__init__.py` 中，通过 `isaacgym_task_map` 字典将任务名称映射到对应的类。

### 3.3 配置系统 (`isaacgymenvs/cfg`)

项目使用 [Hydra](https://hydra.cc/) 进行配置管理，支持分层配置和动态覆盖。

*   **`config.yaml`**: 根配置文件，定义了默认的 `task` 和 `train` 配置，以及全局参数（如 `sim_device`, `rl_device`, `num_envs`）。
*   **`cfg/task/`**: 包含每个环境的具体参数（如物理参数、观测空间大小、奖励函数权重等）。
    *   例如：`task=Ant` 会加载 `cfg/task/Ant.yaml`。
*   **`cfg/train/`**: 包含 RL 算法（主要是 PPO）的超参数（如学习率、Batch size、Gamma 等）。
    *   例如：默认情况下，`Ant` 任务会加载 `cfg/train/AntPPO.yaml`。

### 3.4 强化学习接口 (`rl_games`)

项目默认使用 `rl_games` 库进行训练。`isaacgymenvs` 充当了 Isaac Gym 模拟器和 `rl_games` 算法之间的桥梁。
*   通过 `isaacgymenvs.make()` 创建符合 Gym 接口（Vectorized）的环境。
*   `isaacgymenvs/utils/rlgames_utils.py` 提供了适配器，使 Isaac Gym 的 Tensor 数据可以直接在 GPU 上传递给 `rl_games`，避免了 CPU-GPU 拷贝，实现了端到端的高性能训练。

## 4. 如何添加新任务

1.  **创建任务类**: 在 `isaacgymenvs/tasks` 下新建文件（如 `my_task.py`），继承 `VecTask` 并实现核心方法。
2.  **注册任务**: 在 `isaacgymenvs/tasks/__init__.py` 中导入新类并添加到 `isaacgym_task_map`。
3.  **添加配置**:
    *   在 `isaacgymenvs/cfg/task/` 下添加 `MyTask.yaml`。
    *   在 `isaacgymenvs/cfg/train/` 下添加 `MyTaskPPO.yaml`。
4.  **运行**: 使用 `python train.py task=MyTask` 启动。
