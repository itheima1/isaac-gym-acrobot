# Acrobot 训练策略文档

本文档详细记录了 Acrobot (双连杆倒立摆) 任务的训练策略、奖励机制及关键参数配置。

## 1. 任务描述
目标是将一个双连杆系统（第一关节电机驱动，第二关节被动）从随机初始状态摆动并稳定在直立位置（两个关节角度均为 0）。

## 2. 关键参数配置

### 2.1 物理参数
- **连杆质量**: 0.070 kg (70g) / 每个连杆
  - *配置位置*: [acrobot.urdf](assets/urdf/acrobot.urdf)
- **电机力矩 (Max Effort)**: 1.0 N·m
  - *配置位置*: [Acrobot.yaml](isaacgymenvs/cfg/task/Acrobot.yaml)
  - *说明*: 之前 5.0 仍过大导致螺旋桨式旋转，经计算 0.22 N·m 即可维持水平，故调整为 1.0。
- **环境数量 (Num Envs)**: 4096
  - *说明*: 增加并行环境数量以提高样本效率。
- **仿真参数**: Position Iterations = 8, Velocity Iterations = 1
  - *说明*: 提高物理引擎解算精度，减少穿模和抖动。
- **关节动力学 (Dynamics)**:
  - *说明*: 代码不再强制覆盖 URDF 中的阻尼 (Damping) 和摩擦 (Friction) 设置。
  - *当前设置*: 遵循 URDF (Damping=0.005, Friction=0.01)，确保运动具有真实的物理衰减，避免“永动机”式的异常摆动。

### 2.2 时间参数
- **仿真步长 (dt)**: 0.0166s (60Hz)
- **最大回合长度 (Max Episode Length)**: 500 步
  - *配置位置*: [acrobot.py](isaacgymenvs/tasks/acrobot.py)

## 3. 奖励与惩罚机制 (Reward Function)

**重要更新 (2025-03-13)**: 采用“末端高度”作为核心驱动，彻底解决欧拉角跳变和局部最优问题。

代码实现位于: [acrobot.py](isaacgymenvs/tasks/acrobot.py) 的 `compute_acrobot_reward` 函数。

### 3.1 核心机制 (Tip Height + Hover Bonus)

总奖励 $R_t = 5.0 \cdot R_{\text{height}} + 0.1 \cdot R_{\text{vel}} + 0.05 \cdot R_{\text{action}} + R_{\text{bonus}}$

1.  **末端高度奖励 ($R_{\text{height}}$)**:
    -   **目标**: 直接优化第二根杆末端的全局垂直高度。
    -   **公式**: `(H_tip - H_shoulder) / 0.32`
    -   **范围**: `[-1, 1]` (向下折叠为 -1，垂直向上为 1)。
    -   **权重**: **5.0** (主导项)。
    -   **优势**: 物理意义明确，梯度平滑，无视关节扭曲。

2.  **速度惩罚 ($R_{\text{vel}}$)**:
    -   **公式**: `- (0.5 * v1^2 + 1.0 * v2^2)`
    -   **权重**: **0.1**
    -   **目的**: 抑制轻量化系统（70g/杆）的“直升机效应”，迫使利用重力势能起摆。

3.  **动作惩罚 ($R_{\text{action}}$)**:
    -   **公式**: `- u^2`
    -   **权重**: **0.05**
    -   **目的**: 节能约束，输出平滑力矩，保护电机。

4.  **悬停奖励 ($R_{\text{bonus}}$)**:
    -   **条件**: 高度 > 0.95 (接近最高点) **且** 总角速度 < 1.0 rad/s。
    -   **奖励**: **+10.0** (巨额瞬时奖励)。
    -   **目的**: 解决“静止下垂”的局部最优，强力引导 AI 锁定垂直位置。

### 3.2 重置机制 (Reset)

1.  **旋转限制**: 任意关节绝对角度超过 6.28 (2pi) 时重置。
2.  **超时重置**: 强制在 500 步时重置。

### 3.3 成功标准
-   **Score to Win**: 调整为 **6000**。
-   **理论最大值**: (5.0 * 1.0 + 10.0) * 500 ≈ 7500 分。

## 4. 训练观察指南
-   **Reward 曲线**: 应该稳步上升。
-   **Episode Length**: 初期较短 (因为触发旋转重置)，后期应达到 500 (稳定站立)。

## 5. 关键超参数 (Hyperparameters)
-   **Horizon Length**: 128 (更长的轨迹以捕捉摆动动态)
-   **Minibatch Size**: 32768
-   **Network**: MLP [256, 128] (更大的网络容量)
