# Acrobot 强化学习训练新策略

本文档详细说明了针对 Acrobot（圆形倒立摆）任务的最新训练策略。
该策略放弃了之前容易导致"抖动刷分"的人类直觉奖励，转而采用经过 DeepMind Control Suite 验证的、基于物理势能和稳定性的标准化奖励函数。

---

## 1. 核心目标 (Goal)

训练智能体通过控制第一个关节（Shoulder）的力矩，将两节连杆甩起并**稳定**在最高点。

*   **最高点定义**: 两个连杆都垂直向上。
*   **稳定定义**: 到达最高点后，速度接近于 0，保持静止，不再旋转或抖动。

## 2. 奖励函数设计 (Reward Function)

新的奖励函数由三部分组成：**位置奖励**、**速度惩罚**、**精度奖励**。

### 2.1 位置奖励 (Position Reward) - `rew_dist`

**修正 (2024-03-13 v2)**: 余弦函数在底部（$\pm \pi$）的梯度为零，导致机器人容易陷入"局部最优"，即保持向下不动。
我们改为**线性角度距离** (Linear Angle Distance)，确保在任何位置都有明确的优化方向。

*   **公式**:
    $$
    R_{pos} = 0.5 \times (1 - \frac{|\theta_1|}{\pi}) + 0.5 \times (1 - \frac{|\theta_{tip}|}{\pi})
    $$

*   **代码实现**:
    ```python
    # 1. 归一化角度到 [-pi, pi]
    shoulder_norm = normalize_angle(shoulder_pos)
    tip_norm = normalize_angle(shoulder_pos + elbow_pos)

    # 2. 线性误差 (0 is up, pi is down)
    rew_shoulder = 1.0 - (torch.abs(shoulder_norm) / 3.14159)
    rew_tip = 1.0 - (torch.abs(tip_norm) / 3.14159)

    # 3. 总位置分
    rew_dist = 0.5 * rew_shoulder + 0.5 * rew_tip
    ```

*   **直观解释**:
    *   **梯度恒定**: 无论在底部还是中部，向上移动一点点，奖励就增加一点点。没有"死区"。
    *   这能有效解决机器人"停在下面不动"的问题。

### 2.2 速度惩罚 (Velocity Penalty) - `rew_vel`

**修正 (2024-03-13 v2)**: 移除了可能会阻碍起摆的全局速度惩罚。

1.  **全局微弱阻尼**: $ -0.0001 \times v^2 $ (几乎忽略不计)
2.  **高处强力刹车**: $ -0.05 \times v^2 $ (仅当 `rew_dist > 0.8`)

### 2.3 精度奖励 (Precision Bonus) - `rew_precision`

当机器人非常接近目标状态时，给予额外的奖励，以"锁定"在这个状态，防止微小的漂移。

*   **条件**: 第一杆和第二杆的角度偏差都小于 0.2 弧度（约 11 度）。
*   **奖励**: +1.0 分 (提高奖励权重)。

当机器人非常接近目标状态时，给予额外的奖励，以"锁定"在这个状态，防止微小的漂移。

*   **条件**: 第一杆和第二杆的角度偏差都小于 0.2 弧度（约 11 度）。
*   **奖励**: +0.5 分。

*   **代码实现**:
    ```python
    is_upright = (torch.abs(shoulder_pos) < 0.2) & (torch.abs(elbow_pos) < 0.2)
    rew_precision = 0.5 if is_upright else 0.0
    ```

## 3. 惩罚与重置机制 (Reset Logic)

### 3.1 旋转限制 (Spinning Limit)

为了彻底杜绝"螺旋桨"行为，我们规定关节不能无限旋转。

*   **规则**: 如果第一个关节（Shoulder）旋转超过 1 圈（$\pm 2\pi \approx \pm 6.28$ 弧度），直接结束本局（Reset）。
*   **代码实现**:
    ```python
    limit = 6.28
    reset = torch.where(torch.abs(shoulder_pos) > limit, 1, 0)
    ```
    *(注：新策略中去除了重置时的额外负分惩罚，因为直接打断本身就是一种时间上的惩罚，且 Reset 后从头开始积累奖励效率最低)*

## 4. 总结：为什么新策略更好？

| 旧策略 (Human Intuition) | 新策略 (Standard RL) |
| :--- | :--- |
| **容易刷分**: 只要动就有分 (Swing Incentive)，导致高频抖动。 | **杜绝刷分**: 只有位置高才给分，速度快反而扣分。 |
| **逻辑复杂**: 各种 if-else 分段函数，难以优化。 | **数学平滑**: 连续的 Cosine 函数，梯度清晰，利于神经网络学习。 |
| **目标模糊**: 既要动又要静，产生冲突。 | **目标明确**: 唯一的优化方向就是 **高处 + 静止**。 |

### 预期训练曲线
*   **初期**: 奖励很低，因为大部分时间在下面。
*   **中期**: 偶尔甩上去，奖励出现尖峰。
*   **后期**: 掌握减速技巧，奖励稳定在最大值（约 1.5 分/步 = 1.0位置 + 0.5精度）。
