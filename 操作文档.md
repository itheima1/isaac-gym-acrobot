# Acrobot 任务操作手册

本文档旨在提供 Acrobot 任务的完整操作指南，涵盖环境安装、训练、测试、配置及常见问题排查。

## 1. 环境安装 (Installation)

如果尚未配置环境，请按照以下步骤进行。

### 1.1 前置条件
- Linux 系统 (Ubuntu 18.04/20.04 推荐)
- NVIDIA 显卡驱动已安装
- CUDA Toolkit (推荐 11.x)
- Anaconda 或 Miniconda

### 1.2 安装步骤

1.  **下载 Isaac Gym Preview 4**
    从 [NVIDIA 官网](https://developer.nvidia.com/isaac-gym) 下载 Isaac Gym Preview 4 包。

2.  **安装 Isaac Gym**
    解压并进入 `python` 目录安装：
    ```bash
    tar -xf IsaacGym_Preview_4_Package.tar.gz
    cd isaacgym/python
    pip install -e .
    ```

3.  **安装 IsaacGymEnvs (本项目)**
    进入项目根目录安装依赖：
    ```bash
    cd /home/itheima/IsaacGymEnvs-main
    pip install -e .
    ```

## 2. 模型训练 (Training)

### 2.1 开始新训练
使用默认配置开始训练 Acrobot 任务：
```bash
python train.py task=Acrobot
```
*   **GUI 界面**: 默认会弹出一个 3D 窗口显示训练过程。按 `v` 键可以暂停/恢复渲染（暂停渲染能加快训练速度）。

### 2.2 指定训练轮数
如果你想训练特定的步数（例如 2000 轮），使用 `max_iterations` 参数：
```bash
python train.py task=Acrobot max_iterations=2000
```
*   通常 Acrobot 任务在 1000-2000 轮左右即可收敛。

### 2.3 无界面训练 (Headless)
如果在服务器上运行，或者不需要看画面，开启 headless 模式可以大幅提高速度：
```bash
python train.py task=Acrobot headless=True
```

### 2.4 接着上次进度训练 (Resume)
如果训练中断，或者想在已有模型基础上继续优化：
1.  找到 checkpoint 文件，通常在 `runs/Acrobot/nn/` 目录下，例如 `Acrobot.pth` 或 `last_Acrobot_ep_500.pth`。
2.  使用 `checkpoint` 参数加载：
    ```bash
    python train.py task=Acrobot checkpoint=runs/Acrobot/nn/Acrobot.pth
    ```

### 2.5 覆盖配置参数
你可以在命令行直接修改配置文件中的参数，无需编辑文件：
```bash
# 修改最大力矩和学习率
python train.py task=Acrobot task.env.maxEffort=10.0 train.params.config.learning_rate=1e-3
```

## 3. 测试与评估 (Testing)

### 3.1 运行推理 (Inference)
加载训练好的模型查看效果，**不进行训练**：
```bash
python train.py task=Acrobot test=True checkpoint=runs/Acrobot/nn/Acrobot.pth
```

### 3.2 减少环境数量以便观察
训练时通常使用 4096 个环境并行，测试时为了看清楚动作，建议减少环境数量（例如 16 个）：
```bash
python train.py task=Acrobot test=True num_envs=16 checkpoint=runs/Acrobot/nn/Acrobot.pth
```

### 3.3 录制视频
如果需要录制视频，可以使用 `capture_video=True` (需代码支持) 或直接使用系统录屏软件。

## 4. 关键配置说明 (Configuration)

### 4.1 任务配置 (`isaacgymenvs/cfg/task/Acrobot.yaml`)
控制物理环境、奖励函数和观测空间。
- `env.maxEffort`: 电机最大力矩 (当前: 5.0)。
- `env.resetDist`: 重置环境的阈值。
- `sim.dt`: 仿真时间步长。

### 4.2 训练配置 (`isaacgymenvs/cfg/train/AcrobotPPO.yaml`)
控制 PPO 算法和神经网络结构。
- `network.mlp.units`: 神经网络层级结构 (当前: `[128, 64]`)。
- `learning_rate`: 学习率。
- `gamma`: 折扣因子。
- `max_epochs`: 最大训练轮数。

## 5. 监控与可视化 (Monitoring)

### 5.1 Tensorboard
实时查看奖励曲线、损失函数变化：
```bash
tensorboard --logdir=runs
```
然后打开浏览器访问 `http://localhost:6006`。

### 5.2 常见指标解读
- `rewards/frame`: 平均每帧奖励。越高越好。
- `len/frame`: 平均每回合长度。如果目标是保持直立不倒，这个值越接近最大长度 (500) 越好。

## 6. 常见问题 (Troubleshooting)

### Q1: 训练很久都不收敛 (Reward 只有 -2 左右)
- **原因**: 可能是神经网络太小，或者物理参数太难（力矩不够）。
- **解决**: 
    1. 确保网络结构已更新为 `[128, 64]`。
    2. 检查 `task.env.maxEffort` 是否足够大 (建议 5.0 - 10.0)。
    3. 检查奖励函数是否合理 (参考 `acrobot.py` 中的 `compute_acrobot_reward`)。

### Q2: 机械臂疯狂旋转 (Helicopter mode)
- **原因**: 力矩过大 (`maxEffort`) 或 缺乏阻尼/摩擦。
- **解决**: 
    1. 减小 `maxEffort`。
    2. 在 `acrobot.urdf` 中增加关节的 `damping` 和 `friction`。
    3. 在奖励函数中增加角速度惩罚 (`rew_vel_shoulder`, `rew_vel_elbow`)。

### Q3: 找不到 Checkpoint 文件
- **原因**: 训练出错未保存，或者路径错误。
- **解决**: 使用 `ls -R runs/` 查看实际保存路径。默认只在特定间隔和获得最好成绩时保存。
